init_tk_alslib/[0,1,2]

init_tk_alslib/0 - initialize GUI library, default Tcl interpreter

init_tk_alslib/1 - initialize GUI library, creating Tcl interpreter

init_tk_alslib/1 - initialize GUI library, creating Tcl interpreter

FORMS

init_tk_alslib

init_tk_alslib(Interp)

init_tk_alslib(Interp, Path)

DESCRIPTION

In order to make use of the GUI library routines, one must first initialize the library. This is accomplished one of these predicates. init_tk_alslib/0 initializes a Tcl/Tk interpreter named tcli, and sources the associated Tcl/Tk code into that interpreter. This call is really defined as

init_tk_alslib :- init_tk_alslib(tcli, _).

If Interp is an atom intended to name a Tcl/Tk interpreter, then

init_tk_alslib(Interp, Path)

creates a Tcl/Tk interpreter named Interp, locates the adjunct TclTk code, returns the path to the directory containing that code in Path, and sources that code into Interp.
