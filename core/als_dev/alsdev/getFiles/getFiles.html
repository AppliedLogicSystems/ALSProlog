NAME

getFiles - pop up a dialog box for the user to select multiple files.

SYNOPSIS

package require getFiles

getFiles ? option value ... ?

DESCRIPTION

The procedure getFiles pops up a dialog box for the user to select multiple files. The getFiles command is usually associated with tasks such as adding groups of files to a project.

The following option-value pairs are possible as command line arguments to this command:

-initialdir directory
Specifies that the files in directory should be displayed when the dialog pops up. If this parameter is not specified, then the files in the current working directory are displayed. This option may not always work on the Macintosh. This is not a bug. Rather, the General Controls control panel on the Mac allows the end user to override the application default directory. This option is ignored on the Windows platform.
-parent window
Makes window the logical parent of the dialog. The dialog is displayed on top of its parent window. This option is ignored on the Macintosh platform.
-prompt promptString
Specifies a string to display as a prompt in the dialog box. If this option is not specified, then a default prompt is displayed. This option is ignored on the Macintosh platform.

If the user selects any files, getFiles returns a list containing the full pathnames of the files selected. If the user cancels the operation, the command returns the empty string.

EXAMPLE

package require getFiles

set filelist [getFiles]

if {$filelist != ""} {
    # Process the files ...
}
